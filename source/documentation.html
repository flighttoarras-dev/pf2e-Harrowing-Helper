Harrowing (PF2e) — Party Automation Macro

Version: {{MODULE_VERSION}} | Compatibility: Foundry VTT v13 / PF2e v7.8+

So your GM let you run a Harrowing build!? You don't want to spend the entire session rolling ritual checks and applying effects? Now you can have the AWESOME power of HARROWING RITUAL with all of the boring stuff automated.

Purpose

This macro automates the Harrowing ritual from Pathfinder 2e.

It can perform a Harrowing for:

  • A targeted token

  • A selected party member

  • Multiple party members

  • An entire configured party

The macro:

  • Calculates Harrow checks automatically

  • Detects relevant feats and items

  • Handles Reading the Signs card selection

  • Applies Harrowing effects automatically

  • Supports characters not owned by the player running the ritual

  • Posts a consolidated chat summary for the entire ritual

Setup

Required Modules

  • Advanced Macros

Required Macros

The system uses two macros:

  1. Harrowing

  2. Harrowing Apply Effects

Do not rename either macro. The module automatically configures Harrowing Apply Effects for GM execution on world load.

First Install: After importing the macros from the compendium, manually set Harrowing Apply Effects to Observer permission (right-click the macro → Configure → Default Ownership: Observer). The module will maintain this automatically on subsequent world loads.

Note: A GM must be logged in for the macro to function. The macro will fail to route effects if no GM is present in the world.

Party Configuration

  1. Open each character sheet.

  2. Click the book icon (top left) to obtain the actor UUID.

  3. Add the UUID to the TARGET_PREFS section.

  4. Configure:

    • Name

    • Strategy

    • Suit preference order — use numbers 1–6 per the AoN Harrowing table:

      • 1 = Hammers (Strikes)

      • 2 = Keys (Reflex saves)

      • 3 = Shields (Fortitude saves)

      • 4 = Books (Skill checks)

      • 5 = Stars (Will saves)

      • 6 = Crowns (All other checks)

  5. Repeat for all party members.

Unlisted characters use the fallback preference profile.

Suit Strategies

With the Reading the Signs Harrower 8th level feat the caster may draw 2 and choose 1 for each draw, this allows some strategy in how the suits are selected.

Priority

Always selects the highest-ranked available suit from the character's preference list.

  • Striker: [1, 5, 4, 2, 3, 6]

  • Striker/Skills: [4, 1, 5, 2, 3, 6]

  • Caster: [6, 4, 3, 2, 5, 1]

Spread

Attempts to avoid duplicate suits during a single Harrowing before falling back to preference order.

Fixed Caster (Optional)

The macro can be configured to always use a specific caster by setting:

FIXED_CASTER_ACTOR_UUID

If left blank, the currently selected token is used as the caster.

Setting this is required to use the macro from screens without tokens such as a large area map or world map.

Gameplay Notes

Existing Harrowing Effects

Rules-as-written, a creature cannot benefit from a new Harrowing while any previous Harrowing card effects remain active.

The macro provides a Clear Existing Effects option which removes existing Harrowing effects before applying new ones.

Consuming Harrowing Rerolls

If you need to safely expend Harrowing rerolls outside of combat:

Hammers

  • Any Strike

Books

  • Any skill check

Crowns

  • Seek checks

Shields

  • Drugs

  • Holding breath (be careful on a critical failure)

  • Cantrips

Keys

  • Grab a Ledge

  • Cantrips

Stars

  • Low-level Daze / cast at lower level

  • Certain drugs

  • Frequent Will saves generated by allies

Useful classes include:

  • Bard

  • Psychic

  • Witch

  • Illusion-focused casters

Technical Notes

Harrowing Effect UUID

The macro relies on the PF2e compendium effect:

Spell Effect: Harrowing

If this UUID changes in a future PF2e release, effect application will stop functioning until updated.

Actor Data Dependencies

The macro relies on PF2e actor skill structures including:

  • actor.skills

  • actor.system.skills

Major PF2e data model changes may require updates.

ChoiceSet Handling

The macro pre-seeds PF2e ChoiceSets to avoid effect-selection prompts.

This behavior is PF2e-version sensitive.

If selection dialogs begin appearing unexpectedly after a system update, review:

  • rulesSelections

  • ChoiceSet rule elements

  • Crowns effect handling

Future Maintenance

If Updating the Macro

Perform the following regression tests:

  1. Single target

  2. Multiple targets / whole party

  3. Verify no PF2e ChoiceSet prompts appear

  4. Verify chat card output is correct

  5. Verify effects apply correctly to actors not owned by the player

  6. Verify Clear Existing Effects works correctly

  7. Verify Reading the Signs suit selection works correctly

Companion Macro

The Harrowing macro performs ritual resolution.

The Harrowing Apply Effects macro performs effect application as the GM.

Both macros are required for full functionality.
